## 📊 第三张图：@Cacheable 底层原理（AOP 动态代理）

```
┌─────────────────────────────────────────────────────────────────────────┐
│  你的代码：                                                              │
│  @Service                                                               │
│  public class ProductService {                                          │
│      @Cacheable(cacheNames="product:detail", key="#id")                 │
│      public Product getProductById(Long id) { ... }                     │
│  }                                                                      │
└────────────────────────┬────────────────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────────────────────┐
│  Spring 启动时做了什么？                                                 │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │ 1. 扫描到 ProductService 有@Service                                 │   │
│  │ 2. 准备创建这个 Bean                                              │   │
│  │ 3. 发现 getProductById() 有@Cacheable                                 │   │
│  │ 4. 决定使用 AOP 增强                                                 │   │
│  │ 5. 用 CGLIB 生成一个子类（代理类）                                    │   │
│  └─────────────────────────────────────────────────────────────────┘   │
└────────────────────────┬────────────────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────────────────────┐
│  生成的代理类伪代码：                                                    │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │ public class ProductService$$EnhancerBySpringCGLIB              │   │
│  │        extends ProductService {                                 │   │
│  │                                                                 │   │
│  │    private CacheInterceptor cacheInterceptor;                   │   │
│  │                                                                 │   │
│  │    public Product getProductById(Long id) {                     │   │
│  │        // 这是代理添加的逻辑                                     │   │
│  │        String key = "product:detail::" + id;                    │   │
│  │                                                                 │   │
│  │        // 先去 Redis 查                                           │   │
│  │        Object cached = redisTemplate.get(key);                  │   │
│  │        if (cached != null) {                                    │   │
│  │            return (Product) cached;  // 命中缓存，直接返回        │   │
│  │        }                                                        │   │
│  │                                                                 │   │
│  │        // 没命中，执行原方法                                     │   │
│  │        Product result = super.getProductById(id);               │   │
│  │                                                                 │   │
│  │        // 写入 Redis                                              │   │
│  │        redisTemplate.set(key, result);                          │   │
│  │                                                                 │   │
│  │        return result;                                           │   │
│  │    }                                                            │   │
│  │ }                                                               │   │
│  └─────────────────────────────────────────────────────────────────┘   │
└────────────────────────┬────────────────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────────────────────┐
│  当你调用时：                                                            │
│  productService.getProductById(1L)                                     │
│                                                                         │
│  实际调用的是：                                                          │
│  proxy.getProductById(1L)  ← 代理对象                                    │
└─────────────────────────────────────────────────────────────────────────┘
```

## 📊 第四张图：依赖注入（DI）是怎么回事

```
┌─────────────────────────────────────────────────────────────────────────┐
│  你的代码：                                                              │
│  @RestController                                                        │
│  public class ProductController {                                       │
│      private final ProductService productService;                       │
│                                                                         │
│      public ProductController(ProductService productService) {          │
│          this.productService = productService;                          │
│      }                                                                  │
│  }                                                                      │
└────────────────────────┬────────────────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────────────────────┐
│  传统方式（没有 Spring）：                                                │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │ ProductController controller = new ProductController(           │   │
│  │     new ProductService(                                         │   │
│  │         new ProductMapper(                                      │   │
│  │             new SqlSessionFactory(...)                          │   │
│  │         )                                                       │   │
│  │     )                                                           │   │
│  │ );                                                              │   │
│  │                                                                 │   │
│  │ ❌ 问题：                                                        │   │
│  │ - 耦合严重，改一个依赖要改很多地方                               │   │
│  │ - 不方便测试，无法替换成 Mock 对象                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
└────────────────────────┬────────────────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────────────────────┐
│  Spring 方式（依赖注入）：                                               │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │ Spring 启动时：                                                    │   │
│  │ 1. 扫描到 ProductController                                      │   │

│  │ - 解耦，不需要手动 new                                           │   │
│  │ - 方便测试，可以传入 Mock 对象                                     │   │

│  └─────────────────────────────────────────────────────────────────┘   │
```

## 📊 第五张图：MyBatis 动态代理原理

```
┌─────────────────────────────────────────────────────────────────────────┐
│  你的代码：                                                              │
│  @Mapper                                                                │
│  public interface ProductMapper {                                       │
│      Product findById(@Param("id") Long id);                            │
│  }                                                                      │
└────────────────────────┬────────────────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────────────────────┐
│  Spring 启动时：                                                         │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │ 1. 扫描到 ProductMapper 有@Mapper                                    │   │
│  │ 2. MyBatis 为这个接口生成 JDK 动态代理对象                           │   │
│  │ 3. 代理对象会：                                                  │   │
│  │    - 读取 ProductMapper.xml                                      │   │
│  │    - 找到 namespace="com.example.demo.mapper.ProductMapper"     │   │
│  │    - 记住每个方法对应的 SQL                                       │   │
│  │ 4. 把代理对象放入 Spring 容器                                       │   │
│  └─────────────────────────────────────────────────────────────────┘   │
└────────────────────────┬────────────────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────────────────────┐
│  生成的代理对象伪代码：                                                  │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │ public class ProductMapperProxy                                │   │
│  │        implements ProductMapper {                               │   │
│  │                                                                 │   │
│  │    public Product findById(Long id) {                           │   │
│  │        // 1. 读取 XML 中的 SQL                                       │   │
│  │        String sql = "SELECT * FROM product WHERE id = ?";       │   │
│  │                                                                 │   │
│  │        // 2. 创建 PreparedStatement                             │   │
│  │        PreparedStatement ps = conn.prepareStatement(sql);       │   │
│  │        ps.setLong(1, id);                                       │   │
│  │                                                                 │   │
│  │        // 3. 执行 SQL                                              │   │
│  │        ResultSet rs = ps.executeQuery();                        │   │
│  │                                                                 │   │
│  │        // 4. 映射结果到 Product                                    │   │
│  │        if (rs.next()) {                                         │   │
│  │            Product p = new Product();                           │   │
│  │            p.setId(rs.getLong("id"));                           │   │
│  │            p.setName(rs.getString("name"));                     │   │
│  │            p.setPrice(rs.getDouble("price"));                   │   │
│  │            p.setStock(rs.getInt("stock"));                      │   │
│  │            return p;                                            │   │
│  │        }                                                        │   │
│  │        return null;                                             │   │
│  │    }                                                            │   │
│  │ }                                                               │   │
│  └─────────────────────────────────────────────────────────────────┘   │
└────────────────────────────────────────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────────────────────┐
│  当你调用时：                                                            │
│  productMapper.findById(1L)                                            │
│                                                                         │
│  实际调用的是：                                                          │
│  proxy.findById(1L)  ← 代理对象                                         │
│                                                                         │
│  你完全感知不到代理的存在！                                              │
└────────────────────────────────────────────────────────────────────────┘
```

## 🎯 终极总结：一张表看懂所有知识点

| 层级 | 类名 | 注解 | 作用 | 面试考点 |
|------|------|------|------|----------|
| **启动** | DemoApplication | @SpringBootApplication<br>@EnableCaching | 启动入口，开启自动配置和缓存 | Spring Boot 自动装配原理 |
| **Controller** | ProductController | @RestController<br>@GetMapping<br>@PostMapping | 接收 HTTP 请求，返回 JSON | Spring MVC 工作流程 |
| **Service** | ProductService | @Service<br>@Cacheable<br>@CacheEvict | 业务逻辑，缓存管理 | AOP 动态代理，缓存一致性 |
| **Mapper** | ProductMapper | @Mapper | 数据访问接口（不用实现） | MyBatis 动态代理原理 |
| **XML** | ProductMapper.xml | <mapper><br><select><br><update> | SQL 映射文件 | #{}和${}区别，结果映射 |
| **Model** | Product | 无 | 实体类，对应数据库表 | 字段映射规则 |

## 💡 最最最重要的 3 个问题（必须滚瓜烂熟）

### 1️⃣ Spring Boot 是如何"自动"工作的？
**答案：** 扫描 + 代理
- 扫描所有@Component/@Service/@Controller
- 为带特殊注解的类生成代理对象（AOP）

### 2️⃣ 依赖注入（DI）是什么？
**答案：** Spring 帮你创建对象并注入依赖
- 不用手动 new
- 通过构造函数或@Autowired 注入

### 3️⃣ @Cacheable 底层如何实现？
**答案：** AOP 动态代理
- Spring 生成代理类
- 代理先查缓存，没有才执行原方法
